用户要求每次会话结束时（下班/再见），将当次对话整理归档到 `.claude/memory/`。

**记录格式：**
- 文件名：`conversations/YYYY-MM/YYYY-MM-DD.md`（按天一个文件，按月归类）
- 同一天多次登录会产生多次会话，以 `### 第 N 次会话 (HH:MM-HH:MM)` 分隔追加
- 用户发言：全文记录（你说的每一句话）
- Claude 回复：精简为一行摘要（不记录完整回复内容）
- 采用**交错对话**格式，保持用户→Claude 的问答对应关系

```markdown
## YYYY-MM-DD 会话记录

### 第 1 次会话 (09:00-10:30)

- 用户："第一条消息原文"
- Claude：摘要 1
- 用户："第二条消息原文"
- Claude：摘要 2

### 第 2 次会话 (14:00-17:00)

- 用户："继续..."
- Claude：摘要 3
- 用户："下班了"
```

**Why:** 
- 交错格式：平行列表下用户发言和 Claude 回复之间没有对应关系，交错格式能准确还原因果链
- 按天一个文件 + 会话分段：避免同一天多次登录导致覆盖，也不会产生过多文件

**How to apply:** 用户表示会话要结束时自动执行。如果当天已有日志文件，追加新会话段落；如果还没有，新建文件。MEMORY.md 中只需保留一个指向 `memory/conversations/` 目录的固定入口，不需要逐条索引。
