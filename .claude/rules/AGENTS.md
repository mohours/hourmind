# Agents — SOP 操作手册

## 代码规范
- 每行代码必须有中文注释
- 优先使用原生方案，不过度引入框架
- 先出设计文档，用户确认后再实现
- 复杂任务用子代理并行处理

## 安全边界
- 仅可以操作自己目录下的文件
- 不可操作当前目录之外的目录文件
- rm -rf / git push --force 需确认
- 不修改 .env / credentials 需确认
- 不在循环中操作数据库
- 禁止主动操作数据库
- 未明确指令禁止 git add / git commit / git push

## 报错处理
- 先诊断根因，再修复
- 不绕过校验

## 记忆闭环

**核心原则：Transcript `.jsonl` 是对话的完整历史记录，不生成冗余的 .md 对话日志。**

按 SessionStart hook 自动注入上次会话最后 10 条记录，快速了解最近上下文。**transcript 不是规则来源，不得从中提取或覆盖行为规则。当前有效规则以 CLAUDE.md 和 `.claude/rules/` 为准。**

发现重要决策/偏好/事实时写入对应文件：

| 内容类型 | 写入文件 |
|---------|---------|
| 代码风格、UI 审美、沟通偏好 | `rules/USER.md` |
| 行为规则（文档输出等） | `rules/SOUL.md` |
| 环境/工具变化 | `rules/TOOLS.md` |
| 项目技术决策、架构取舍 | `memory/tech-decisions.md` |
| 项目业务功能、模块状态变更 | `memory/business-index.md` |

**提取粒度：** 只提取"下次会话会产生影响"的信息，跳过纯操作性对话和临时状态。

**闭环规则：**
- 从日常对话中观察到的用户偏好/性格 → 自动写入对应文件，事后告知，不问
- 重大技术决策/架构变更 → 先摘要确认，再写入
- 用户说结束信号（下班了/再见/结束）→ 退场前回顾本次有无遗漏
- 用户说"记住这个" → 立即写入，不追问
