# HourMind - API Key 管理 详细产品需求文档（PRD）

## 1. 页面基本信息

**页面名称**：API Key 管理（API Keys Management）  
**页面定位**：系统 AI 能力的核心控制中心  
**核心目标**：让用户安全、便捷、高效地管理系统所使用的各大 AI 服务商 API Key，提供透明的用量监控和智能路由能力。

**设计风格**：Quantum Glass（量子玻璃）—— 强调安全感、科技感和专业性。

---

## 2. 用户价值

- 一站式管理所有 AI 厂商的 Key，无需记住多个平台
- 实时掌握各 Key 的使用情况和费用，避免超支
- 快速测试和排查 Key 问题
- 支持智能模型路由，自动选择最优模型
- 最高级别的 Key 安全保护

---

## 3. 核心功能需求

### 3.1 Key 的 CRUD 操作
- **添加 Key**：支持主流厂商（OpenAI、Anthropic/Claude、DeepSeek、Google Gemini、xAI/Grok、Moonshot、通义千问等）
- **批量导入**：支持一次性添加多个 Key
- **编辑**：修改别名、使用场景标签
- **删除**：二次确认 + 软删除（可恢复）
- **禁用/启用**：临时关闭某个 Key

### 3.2 Key 测试功能
- 一键测试连通性
- 显示响应速度（毫秒）、可用模型列表、限额信息
- 测试历史记录（最近 7 次）

### 3.3 用量监控与统计
- 每个 Key 的实时 Token 消耗（今日/本月）
- 预估费用展示（支持不同币种）
- 使用趋势折线图
- 全局用量仪表盘（所有 Key 汇总）

### 3.4 智能路由规则
- 默认模型设置
- 规则引擎：根据任务类型、复杂度、成本智能选择模型
  - 示例规则：
    - 复杂推理 → Claude 4
    - 快速回复 → DeepSeek
    - 代码生成 → GPT-4o / DeepSeek
- 手动优先级排序
- Fallback 机制（主模型失败自动切换）

### 3.5 安全与隐私保护
- Key 加密存储（绝不明文保存）
- 前端完全不暴露任何 Key
- 操作日志记录（谁在什么时间操作了哪个 Key）
- Key 部分脱敏显示（仅显示后 6 位）

---

## 4. 页面布局与 UI 细节（Quantum Glass）

**整体布局**：
- 左侧导航栏（统一）
- 中间主内容区（Key 列表）
- 右侧：选中 Key 的详细信息面板

**详细设计**：

**顶部栏**：
- 大标题 “API Key 管理”
- 醒目“+ 添加新 API Key” 量子青按钮
- 全局搜索框（按平台、别名搜索）
- 统计概览（总 Key 数、活跃 Key 数、本月总消耗）

**Key 列表区**（玻璃卡片形式）：
- 每张卡片包含：
  - 左侧：平台 Logo + 名称
  - 中间：别名 + 创建时间 + 使用场景标签
  - 右侧：状态指示灯（正常/警告/失效）+ 用量小环形图 + 操作按钮（测试、编辑、删除）

**右侧详情面板**：
- Key 详细信息
- 用量趋势图
- 路由规则设置
- 测试记录
- 安全信息（最后使用时间、创建 IP 等）

**添加/编辑弹窗**：
- 玻璃模态框
- 选择平台（带搜索）
- 输入 Key（支持粘贴）
- 填写别名 + 使用场景
- 立即测试选项

---

## 5. 交互与动效

- **添加 Key**：成功后卡片带有量子青入场动画
- **测试过程**：按钮变为加载状态 + 进度提示
- **卡片悬停**：轻微上浮 + 辉光增强
- **状态变化**：失效 Key 显示橙色警告辉光
- **拖拽排序**：支持调整 Key 优先级

---

## 6. 与其他模块的关联

- **智能对话**：直接读取此处配置的可用模型
- **模型设置**：路由规则在此页面配置
- **仪表盘**：显示全局 Key 用量摘要
- **系统设置**：整体安全策略配置

---

## 7. 非功能需求（安全为最高优先级）

- **安全性**：
  - 所有 Key 使用 AES-256 加密存储
  - 仅后端可访问真实 Key
  - 支持主密码/生物识别解锁（Electron 版）
- **性能**：页面加载 < 800ms
- **兼容性**：支持未来新增 AI 厂商
- **审计**：所有 Key 操作均有日志
- **容灾**：Key 备份与恢复机制

---

## 8. 开发优先级（MVP）

**Phase 1（必须）**：
1. Key 的增删改查
2. 平台选择与添加
3. 基础测试功能
4. Key 列表展示（玻璃卡片）

**Phase 2**：
5. 用量统计与图表
6. 安全加密存储
7. 模型路由规则

**Phase 3**：
8. 批量操作
9. 高级监控告警（额度预警）
10. 操作日志查看
