# HourMind 产品作业流程

## 1. client 模块是什么

client 就是浏览器里看到的前端界面。打开 `http://localhost:5173` 看到的一切 — 登录页、仪表盘、聊天窗口、Key 管理页 — 全是 client 模块渲染的。

server 是背后处理数据的后端服务，AI 厂商是外部调用的。整套流程就是：**你在浏览器里打字 → 后端加密传输给 AI → AI 逐字吐回来 → 实时显示在屏幕上**。

## 2. 完整操作流程

### 阶段 1：环境准备（首次）
1. 启动后端 `npm run dev` → 端口 3000
2. 启动前端 `npm run dev` → 端口 5173
3. 浏览器打开 `http://localhost:5173`

### 阶段 2：首次设置
1. 输入密码（两次确认）
2. 后端用 bcrypt 哈希密码，存入 AppConfig 表
3. 返回 JWT Token（24h 有效），前端存 localStorage
4. 跳转仪表盘首页

### 阶段 3：日常登录
1. 输入密码
2. 后端 bcrypt.compare 验证
3. 返回 JWT Token
4. 进入仪表盘

### 阶段 4：添加 API Key（仅首次，或添加新 Key）
1. 点击"+ 添加新 Key"
2. 选择 AI 厂商（OpenAI / DeepSeek / Claude 等 7 家）
3. 粘贴 API Key 明文
4. 后端用 AES-256-GCM 加密，存入 SQLite
5. 前端永远只拿到 keySuffix（后 6 位）
6. 可选：测试连通性 → 后端 GET 厂商 /models → 记录延迟 → 状态=active/error

### 阶段 5：智能对话（核心功能）
1. 新建会话 → 输入消息
2. 后端解密 Key → 调厂商 API（POST /chat/completions, stream=true）
3. 厂商逐 token 返回 → WebSocket ctx.push('stream_chunk') 实时推前端
4. 前端逐字追加到 AI 消息气泡（打字机效果）
5. 流结束 → 更新消息内容 + Token 用量 → 存 TokenUsageRecord

### 阶段 6：仪表盘（首页概览）
1. 页面加载 → dashboard.summary 聚合查询
2. 今日 Token 消耗 / 对话次数 / 费用
3. 模型使用排行（前 5 名）
4. 最近 6 条对话
5. 快速入口卡片（新对话 / Key 管理 / 知识库 / 待办事项）

### 阶段 7：历史记录
1. 查看所有历史对话列表
2. 搜索、筛选（状态/模型/标签）
3. 打开对话 → 加载消息 → 跳转聊天页继续对话
4. AI 一键生成会话摘要（调厂商 API 非流式）
5. 批量操作：导出(MD/JSON)、打标签、归档、删除
6. 恢复已删除对话

### 阶段 8：系统设置
1. 默认 AI 模型
2. 模型温度（0~1）
3. 粒子背景开关
4. 辉光强度（low/medium/high）
5. 保存 → settings.update → upsert AppConfig 表

## 3. 日常使用循环路径

```
登录 → 仪表盘 → 新建对话 → 输入消息 → AI 流式回复
  │       │          │
  │       │          └→ 切换模型 → 继续对话
  │       │
  │       └→ 检查 Token 用量 → 管理 Key
  │
  └→ 历史记录 → 搜索旧对话 → 继续聊 / AI 摘要 / 导出
```

## 4. 底层数据流（前后端协作）

```
浏览器 (client)                  Node.js 后端 (server)             AI 厂商 API
─────────────                    ──────────────────               ────────────

Vue 组件                         wsRouter 分发
    │                                │
    ├─ wsClient.send() ──WebSocket──→ handleMessage()
    │                                │
    │                                ├─ handler 执行业务逻辑
    │                                │   └─ Prisma 操作 SQLite
    │                                │
    │                                ├─ (对话时) streamChat() ──HTTP──→ 厂商
    │                                │   └─ read stream ── ctx.push() ──→ 前端
    │                                │
    │◀── 响应帧 / 推送帧 ──WebSocket──┤
    │
Pinia Store 更新状态
    │
Vue 响应式刷新 UI
```

### 帧格式

**请求帧（前端→后端）：**
```json
{ "id": "req_001", "type": "request", "action": "keys.list", "payload": {} }
```

**响应帧（后端→前端）：**
```json
{ "id": "req_001", "type": "response", "success": true, "data": [...] }
```

**推送帧（后端→前端，流式对话）：**
```json
{ "type": "stream_chunk", "message_id": "yyy", "chunk": "你好" }
{ "type": "stream_end", "message_id": "yyy", "token_count": 524 }
```

### 关键区别

| 帧类型 | 有 id 吗 | 说明 |
|--------|---------|------|
| 请求帧 | 有 | 前端发起，等待对应 id 的响应 |
| 响应帧 | 有 | 后端回复，根据 id 匹配请求 |
| 推送帧 | 没有 | 后端主动推送（流式对话），不参与匹配 |

## 5. 技术栈总览

| 层 | 技术 | 说明 |
|----|------|------|
| 前端框架 | Vue 3 + TypeScript | 组件化 UI |
| 前端构建 | Vite | 开发服务器 + 打包 |
| 前端状态 | Pinia | 全局数据管理 |
| 后端框架 | Express | HTTP 服务 |
| 后端通信 | WebSocket (ws) | 实时双向通信 |
| 数据库 | SQLite + Prisma | 本地文件数据库 + ORM |
| 认证 | JWT + bcrypt | Token 鉴权 + 密码哈希 |
| 加密 | AES-256-GCM | API Key 加密存储 |
| AI 调用 | 直接 HTTP | 调厂商 /chat/completions |
| Markdown | marked + highlight.js | AI 回复渲染 |
